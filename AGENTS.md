# AGENTS.md - Coding Guidelines

## Build Commands

```bash
# Development server (runs on port 5173)
pnpm dev

# Production build (outputs to dist/)
pnpm build

# Preview production build
pnpm preview
```

**Note:** This project uses `pnpm` (not npm). Install dependencies with `pnpm install`.

## Project Structure

- **Source:** `src/` - HTML files, JS modules, CSS
- **Assets:** `assets/` - Images, fonts, media files
- **Public:** `public/` - Static files copied as-is
- **Build output:** `dist/` - Generated by Vite

## Tech Stack

- **Build Tool:** Vite with multi-page setup
- **Styling:** Tailwind CSS v3 + custom CSS. Prefer tailwind CSS if you can
- **PostCSS:** Autoprefixer for vendor prefixes
- **JS:** ES6 modules, vanilla JavaScript
- **Components:** Custom Web Components with Shadow DOM
- **Graphics:** p5.js for interactive canvas elements

## Code Style Guidelines

### JavaScript

- Use ES6 modules (`type: "module"` in package.json)
- Prefer `const` and `let` over `var`
- Use double quotes for strings
- Template literals with tagged comments for syntax highlighting:
  ```javascript
  const html = /* html */ `<div>...</div>`;
  const css = /* css */ `.class { ... }`;
  ```
- Arrow functions for callbacks: `(x) => x * 2`
- Async/await for asynchronous operations
- Error handling with try/catch for async operations

### Naming Conventions

- **Variables/Functions:** camelCase (`createCards`, `isLoading`)
- **Classes:** PascalCase (`HeaderComponent`, `FooterComponent`)
- **Custom Elements:** kebab-case (`header-component`, `footer-component`)
- **Files:** Lowercase with hyphens (`scroll-to-top.js`, `global.styles.css`)
- **CSS Classes:** Tailwind utilities preferred; custom classes use kebab-case

### HTML

- Semantic HTML5 elements
- Custom elements for reusable components:
  ```html
  <header-component></header-component> <footer-component></footer-component>
  ```
- Lazy loading for images: `loading="lazy"`
- Accessible attributes: `alt` text required for images

### CSS/Styling

- **Primary:** Tailwind CSS utility classes
- **Custom CSS:** In `global.styles.css` using `@layer` directives
- Custom colors defined in `tailwind.config.js`
- Responsive breakpoints: `sm:`, `md:`, `lg:`, `xl:`, `2xl:`
- Custom animations in CSS with `@keyframes`

### Web Components

- Use Shadow DOM with `mode: "open"`
- Define custom elements: `customElements.define("name", Class)`
- Lifecycle methods: `connectedCallback`, `disconnectedCallback`
- Encapsulated styles within component templates

### Imports

```javascript
// ES6 imports (absolute paths from src/)
import { defineConfig } from "vite";
import { resolve } from "path";

// Vite handles module resolution
```

### Error Handling

- Check for DOM elements before manipulation
- Console.error for debugging: `console.error("Message:", selector)`
- Graceful fallbacks for missing elements

### Formatting

- Prettier is configured (v3.6.2)
- Run: `npx prettier --write .` to format all files
- No explicit lint command configured
